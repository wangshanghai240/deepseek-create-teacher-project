# 教师管理平台 - 腾讯云服务器部署教程

---

## 一、准备工作

### 1.1 需要的东西
- 一台腾讯云服务器（CentOS 7/8 或 Ubuntu 20.04+）
- 服务器的 **公网 IP**、**用户名**、**密码** 或 **SSH 密钥**
- MySQL 数据库（可用腾讯云云数据库，或继续用现有的 `mysql.sqlpub.com`）

---

## 二、连接服务器

### Windows 用 PowerShell 连接（推荐）
```powershell
# 在 Windows 搜索 "PowerShell"，打开后输入：
ssh root@你的服务器IP
# 然后输入密码
```

### 用 VS Code 连接（更方便）
1. 安装扩展：**Remote - SSH**
2. 按 `F1` → 输入 `Remote-SSH: Connect to Host`
3. 输入：`ssh root@你的服务器IP`
4. 连接成功后就可以直接在 VS Code 里操作远程服务器

---

## 三、安装 Node.js

### CentOS 系统
```bash
# 安装 Node.js 18.x LTS
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs

# 验证安装
node -v
npm -v
```

### Ubuntu 系统
```bash
# 安装 Node.js 18.x LTS
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt-get install -y nodejs

# 验证安装
node -v
npm -v
```

---

## 四、安装 PM2（进程管理，推荐）

PM2 可以让 Node.js 服务在后台持续运行，即使关闭终端也不会停止。

```bash
npm install -g pm2

# 验证
pm2 -v
```

---

## 五、上传项目代码

### 方法一：用 Git（推荐）

```bash
# 如果代码在 GitHub/Gitee 上
yum install -y git   # CentOS
# apt-get install -y git   # Ubuntu

git clone https://github.com/你的用户名/teacher_project.git
cd teacher_project
```

### 方法二：用 SCP 上传

```bash
# 在本机 PowerShell 中执行（不是在服务器上）
# 将整个项目上传到服务器
scp -r F:\VScode\teacher_project root@你的服务器IP:/root/
```

### 方法三：用 VS Code Remote-SSH
连接上服务器后，可以直接拖拽文件到 VS Code 的远程文件管理器里。

## 六、安装项目依赖

```bash
# 进入服务器上的项目目录
cd /root/teacher_project/server

# 安装后端依赖
npm install

# 如果安装慢，可以用淘宝镜像
npm install --registry=https://registry.npmmirror.com
```

---

## 七、配置防火墙（开放端口）

```bash
# 开放 5000 端口（后端 API）
firewall-cmd --zone=public --add-port=5000/tcp --permanent
firewall-cmd --reload

# 如果是 Ubuntu，用 ufw
# ufw allow 5000
```

同时需要在 **腾讯云控制台 → 安全组** 中添加**入站规则**，开放端口 **5000**（以及后续 80/443）。

---

## 八、启动后端服务

```bash
cd /root/teacher_project/server

# 用 PM2 启动（推荐，后台运行）
NODE_ENV=production pm2 start index.js --name teacher-api

# 保存 PM2 进程列表（服务器重启后自动启动）
pm2 save
pm2 startup

# 查看运行状态
pm2 status
pm2 logs teacher-api
```

## 九、配置 Nginx 反向代理（可选但推荐）

Nginx 可以将 80/443 端口转发到 5000，方便后续绑定域名和配置 HTTPS。

```bash
# 安装 Nginx
yum install -y nginx   # CentOS
# apt-get install -y nginx   # Ubuntu

# 启动 Nginx
systemctl start nginx
systemctl enable nginx
```

创建 Nginx 配置：

```bash
vim /etc/nginx/conf.d/teacher.conf
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name 你的服务器IP或域名;

    # 前端静态文件（需要先构建前端）
    root /root/teacher_project/vue-app/dist;
    index index.html;

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Vue Router 的 SPA 回退
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

```bash
# 检查配置并重载
nginx -t
systemctl reload nginx
```

## 十、构建前端并部署

在**本地电脑**上构建前端：

```bash
cd F:\VScode\teacher_project\vue-app
npm run build
```

然后将生成的 `dist` 文件夹上传到服务器：

```bash
scp -r F:\VScode\teacher_project\vue-app\dist root@你的服务器IP:/root/teacher_project/vue-app/dist
```

---

## 十一、更新前端 API 地址

在本地修改 `vue-app/vue.config.js`，将代理目标改为线上地址：

```javascript
// 本地开发时用代理
devServer: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://你的服务器IP:5000',  // 改为服务器地址
      changeOrigin: true
    }
  }
}
```

在 `vue-app/src/network.js` 或 api 请求文件中，生产环境 URL 应改为：

```javascript
// 根据环境使用不同 API 地址
const BASE_URL = process.env.NODE_ENV === 'production'
  ? 'http://你的服务器IP:5000'  // 或你的域名
  : '/api'
```

## 十二、访问你的应用

- **后端 API**：`http://你的服务器IP:5000`
- **前端页面**（如果配置了 Nginx）：直接访问 `http://你的服务器IP`
- **查看日志**：`pm2 logs teacher-api`

## 常用运维命令

```bash
# PM2 常用命令
pm2 status              # 查看所有进程
pm2 logs teacher-api    # 查看日志
pm2 restart teacher-api # 重启
pm2 stop teacher-api    # 停止
pm2 delete teacher-api  # 删除进程

# 查看端口占用
netstat -tlnp | grep 5000

# 查看系统资源
top
df -h
```
